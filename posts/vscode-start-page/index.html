VSCode 이전 작업폴더 기억하지 않고 기본 시작화면 사용하기

2022년 11월 4일 금요일
  • vscode

비주얼 스튜디오 코드에서는 창을 닫고 열면 이전에 작업하던 작업폴더를 기억하고 가져온다.
ssh 원격폴더로 주로 사용하는데 딜레이도 있고 해서 빈창에 시작화면으로 설정.

설정에서 restore windows 검색하면 관련 설정을 빠르게 찾을 수 있다.
설정 단축키는 Ctrl + ,

Windows: Restore Windows

처음 시작한 후 창이 다시 열리는 방식을 제어합니다. 이 설정은 애플리케이션이 이미 실행 중인 경우에는 아무런 영향을 주지 않습니다.

  • all(기본값)
    폴더, 작업 영역 또는 파일이 열려 있지 않은 한(예: 명령줄에서) 모든 창을 다시 엽니다.
  • preserve
    항상 모든 창을 다시 엽니다. 폴더 또는 작업 영역이 열려 있는 경우(예: 명령줄에서) 이전에 열리지 않은 한 새 창으로 엽니다. 파일이 열려 있는 경우 파일이 복원된 창 중 하나에서 열립니다.
  • folders
    폴더, 작업 영역 또는 파일이 열려 있지 않은 한(예: 명령줄에서) 열린 폴더 또는 작업 영역이 있는 모든 창을 다시 엽니다.
  • one
    폴더, 작업 영역 또는 파일이 열려 있지 않은 한(예: 명령줄에서) 마지막 활성 창을 다시 엽니다.
  • none
    창을 다시 열지 않습니다. 폴더 또는 작업 영역이 열려 있지 않은 한(예: 명령줄에서) 빈 창이 표시됩니다.
